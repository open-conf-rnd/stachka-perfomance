# TG Mini App — шаблон

Стартовый шаблон для разработки **Telegram Mini Apps** на React, TypeScript и Vite. Включает Docker-окружение с HTTPS (nginx + SSL) для локальной разработки и тестирования в клиенте Telegram, production-сборку и интеграцию с [@telegram-apps/sdk-react](https://github.com/Telegram-Mini-Apps/sdk).

Шаблон для быстрого старта локальной и production-разработки **React + TypeScript + Vite** приложения под **Telegram Mini Apps**.

- **React 18 + TypeScript + Vite** — сборка и dev-сервер
- **Docker** — локальная разработка (Vite + nginx) и production (nginx со статикой)
- **Nginx** — HTTPS-прокси для локальной разработки (Telegram требует HTTPS)
- **@telegram-apps/sdk-react** — SDK для Mini App

## Быстрый старт

### 1. Зависимости

```bash
npm install
```

### 2. Домен и SSL для HTTPS (обязательно для теста в Telegram)

Telegram и BotFather принимают только валидный HTTPS-URL; ссылка на localhost не подходит для отображения и теста в клиенте Telegram. Нужен домен (например, туннель ngrok/cloudflared или свой).

1. В `.env` задайте домен, на который будет проксироваться HTTPS:
   ```bash
   DOMAIN=your-subdomain.local
   ```

2. Сгенерируйте самоподписанный сертификат **для этого домена**:
   ```bash
   chmod +x scripts/gen-ssl.sh
   ./scripts/gen-ssl.sh
   ```
   Скрипт читает `DOMAIN` из `.env` (или первый аргумент: `./scripts/gen-ssl.sh my.domain.com`). В `docker/nginx/ssl/` появятся `cert.pem` и `key.pem`.

#### Локальный домен без ngrok и Cloudflare

Если не хотите использовать туннели, можно поднять свой домен только на этой машине (например `udwgrosh.local`). Так можно разрабатывать и проверять приложение в браузере по HTTPS. **В клиенте Telegram** (телефон, десктоп) этот адрес открыть нельзя — его видит только ваш компьютер; для теста внутри Telegram нужен туннель или реальный домен.

**Шаг 1. Выбрать имя домена**

Выберите имя вида `udwgrosh.local` или `udwgrosh.test`.

- **Зачем:** суффиксы `.local` и `.test` зарезервированы для локального использования и не уходят в интернет. Так вы не пересечётесь с реальными доменами и сможете спокойно использовать HTTPS с самоподписанным сертификатом.

**Шаг 2. Прописать домен на этой машине**

Добавьте запись в файл hosts, чтобы имя домена указывало на ваш компьютер:

- **macOS / Linux:** отредактируйте `/etc/hosts` с правами администратора:
  ```bash
  sudo nano /etc/hosts
  ```
  Добавьте строку (подставьте свой домен):
  ```
  127.0.0.1 udwgrosh.local
  ```
  Сохраните файл.

- **Зачем:** без этой записи браузер не знает, куда идти по адресу `udwgrosh.local`. Запись в hosts говорит: «запросы на этот домен отправляй на 127.0.0.1», то есть на nginx в Docker на вашей машине.

**Шаг 3. Указать домен в проекте**

В файле `.env` в корне проекта задайте тот же домен:

```env
DOMAIN=udwgrosh.local
```

- **Зачем:** nginx и скрипт генерации SSL используют переменную `DOMAIN`: nginx слушает запросы на это имя, а сертификат должен быть выдан именно для него, иначе браузер покажет ошибку несоответствия домена.

**Шаг 4. Сгенерировать SSL-сертификат**

Из корня проекта выполните:

```bash
chmod +x scripts/gen-ssl.sh
./scripts/gen-ssl.sh
```

- **Зачем:** Telegram Mini App открывается только по HTTPS. Скрипт создаёт самоподписанный сертификат для `DOMAIN` из `.env` и кладёт его в `docker/nginx/ssl/`. Nginx в Docker будет отдавать приложение по HTTPS с этим сертификатом.

**Шаг 5. Запустить приложение**

```bash
docker compose -f docker/docker-compose.yml up --build
```

Откройте в браузере: **https://udwgrosh.local** (или тот домен, что указали).

- **Зачем:** Docker поднимает Vite и nginx; nginx принимает запросы на ваш домен по 443 порту и проксирует их на dev-сервер, так что у вас есть HTTPS и горячая перезагрузка (HMR).

**Почему браузер пишет «Не защищено»:** используется самоподписанный сертификат — браузер ему по умолчанию не доверяет. Это нормально для локальной разработки.

- **Просто зайти на сайт:** на странице предупреждения нажмите «Дополнительно» → «Перейти на udwgrosh.local (небезопасно)» (или аналогичную ссылку). Сайт откроется, соединение по HTTPS есть, данные между браузером и вашим компьютером шифруются.
- **Важно:** если сертификат не добавлен в доверенные, в **браузере** приложение будет открываться (можно перейти по ссылке), а в **Telegram Mini App** — нет. Клиент Telegram не даёт обойти предупреждение о недоверенном сертификате, в отличие от браузера. Поэтому для проверки мини-приложения внутри Telegram (например, по туннелю на ваш компьютер) сертификат нужно добавить в доверенные.
- **Убрать предупреждение и показывать замочек:** добавьте сертификат в доверенные. На **macOS**:
  1. Из корня проекта выполните:
     ```bash
     chmod +x scripts/trust-ssl-macos.sh
     ./scripts/trust-ssl-macos.sh
     ```
     Введите пароль администратора (если сертификат уже в ключнице, скрипт сообщит об этом и выведет инструкцию ниже).
  2. Выставьте доверие вручную в «Связка ключей»:
     - Откройте **Связка ключей** (Keychain Access).
     - Слева выберите **Система** (System).
     - В поиске введите ваш домен, например **udwgrosh.local**.
     - Дважды нажмите на найденный **сертификат** (не ключ).
     - Раскройте блок **Доверие** (Trust).
     - Для пункта «При использовании этого сертификата» выберите **Всегда доверять** (Always Trust).
     - Закройте окно и введите пароль, если потребуется.
  3. Полностью закройте браузер и откройте снова, зайдите на `https://udwgrosh.local` — должно отображаться защищённое соединение (замочек).

**Если появляется ошибка «This host is not allowed»:** по умолчанию Vite принимает запросы только с `localhost`. Когда вы заходите по кастомному домену (`udwgrosh.local`), nginx передаёт этот домен в заголовке `Host`, и Vite его отклоняет. В этом шаблоне в `vite.config.ts` уже задано `server.allowedHosts: true`, так что любой хост (в том числе ваш локальный домен) разрешён. Если ошибка всё же возникла, проверьте, что в конфиге есть эта опция, и перезапустите контейнеры.

**Итог:** приложение доступно по `https://udwgrosh.local` только на этом компьютере. Для проверки в самом Telegram (на телефоне или в десктоп-клиенте) нужен публичный HTTPS-URL: туннель (ngrok, cloudflared и т.п.) или развёрнутый домен.

### 3. Запуск

**Локальная разработка (без Docker):**

```bash
npm run dev
```

Приложение: http://localhost:5173  
Для проверки в Telegram нужен HTTPS — используйте Docker или туннель (ngrok и т.п.).

**Локальная разработка с HTTPS (Docker):**

```bash
docker compose -f docker/docker-compose.yml up --build
```

- Приложение за nginx: **https://&lt;DOMAIN&gt;** (из `.env`). При самоподписанном сертификате браузер покажет предупреждение — это нормально.
- Тот же dev-сервер доступен по **http://localhost:5173** (порт проброшен из контейнера). Для Mini App и HTTPS используйте адрес через nginx (п. выше).
- Если используете туннель (ngrok и т.п.), поднимите его на порт 443 и укажите выданный домен в `DOMAIN`; после смены домена заново выполните `./scripts/gen-ssl.sh`.
- Vite HMR работает через nginx.

**Если «localhost» не открывается (ERR_CONNECTION_REFUSED):**
- Убедитесь, что контейнеры запущены: `docker compose -f docker/docker-compose.yml up --build` (из корня проекта). Ждите сообщение Vite о готовности.
- Открывайте **https://localhost** (порт 443) или **https://udwgrosh.local** (если настроили локальный домен), а не `http://localhost:5173`, если хотите работать через HTTPS. Либо используйте **http://localhost:5173** — порт теперь проброшен.
- Проверьте написание: `localhost` (с буквой **t**), не `loalhost`; для порта — `localhost:5173` (с двоеточием).

**Production-сборка (Docker):**

```bash
docker compose -f docker/docker-compose.yml -f docker/docker-compose.prod.yml up --build -d
```

Сервис слушает порт 80 и раздаёт статику из `dist`.

## Переменные окружения

Скопируйте `.env.example` в `.env` и заполните:

- `TG_BOT_API` — токен бота от [@BotFather](https://t.me/BotFather)
- `TG_DOMAIN` — username бота (без @)
- `DOMAIN` — домен для HTTPS при разработке (nginx и SSL). Нужен валидный URL для теста в клиенте Telegram; подойдёт домен туннеля (ngrok, cloudflared) или свой. После смены — заново запустите `./scripts/gen-ssl.sh`.

Для production можно задать `VITE_APP_URL` (публичный URL Mini App).

## Подключение Mini App в Telegram

1. [@BotFather](https://t.me/BotFather) → бот → **Bot Settings** → **Menu Button** или **Configure Mini App**.
2. Укажите URL:
   - Разработка: `https://<DOMAIN>` — тот же домен, что в `.env` (туннель или ваш домен). localhost в BotFather не принимается.
   - Production: `https://your-domain.com`.
